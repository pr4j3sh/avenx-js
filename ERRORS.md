# Avenx-JS Error Code Reference

This document provides a detailed list of all error and warning codes generated by the Avenx-JS framework during both compilation and runtime. 

Each error is prefixed with a code like `[AVX_C01]` or `[AVX_R01]`, which helps identify whether the error occurred during the **Compiler (C)** phase or the browser **Runtime (R)** phase.

---

## Index of Error Codes

| Code | Type | Scope | Description |
|---|---|---|---|
| [AVX_C01](#avx_c01) | Error Log | Compiler | Failed to create the build output directory. |
| [AVX_C02](#avx_c02) | Error Log | Compiler | The source directory (`src`) does not exist. |
| [AVX_R01](#avx_r01) | Exception | Runtime | Mount target container element not found in DOM. |
| [AVX_R02](#avx_r02) | Exception | Runtime | Requested Page component not registered. |
| [AVX_R03](#avx_r03) | Exception | Runtime | Requested child Component not registered. |
| [AVX_R04](#avx_r04) | Warning | Runtime | Circular computed property reference loop detected. |
| [AVX_R05](#avx_r05) | Warning | Runtime | Computed property evaluation execution failed. |
| [AVX_R06](#avx_r06) | Warning | Runtime | Navigation guard explicitly denied the route transition. |
| [AVX_R07](#avx_r07) | Error Log | Runtime | Unhandled error thrown inside navigation guard `canActivate()`. |
| [AVX_R08](#avx_r08) | Warning | Runtime | Interpolation expression `{{ ... }}` template rendering failed. |
| [AVX_R09](#avx_r09) | Error Log | Runtime | Inline event action statement execution failed. |
| [AVX_R10](#avx_r10) | Throw Exception | Runtime | Shared state bridge with the same name already registered. |
| [AVX_R11](#avx_r11) | Throw Exception | Runtime | State mutation detected during the update/render lifecycle. |

## Compiler Errors (`AVX_C*`)

### AVX_C01
* **Message:** `Could not create dist directory at "{distDir}".`
* **Type:** Error Log (Node execution halts or falls back)
* **Cause:** The compiler attempted to create the build output directory (`dist`) but did not have appropriate filesystem write permissions.
* **Resolution:** Ensure the running process has permission to write in the project root directory. Run with appropriate privileges if necessary.

### AVX_C02
* **Message:** `"src" directory not found at "{srcDir}". Run "avenx init" to scaffold a project.`
* **Type:** Error Log
* **Cause:** The compiler was executed in a directory that does not contain a `src/` folder.
* **Resolution:** 
  1. Make sure you are running the command from your project root.
  2. If starting a new project, run `npx avenx init` to scaffold the recommended project directory structure.

## Runtime Errors (`AVX_R*`)

### AVX_R01
* **Message:** `Mount target selector "{selector}" was not found in the DOM.`
* **Type:** Throw Exception (`AvenxError`)
* **Cause:** The application or component was initialized with a target DOM selector (e.g. `target: '#app'`), but no element matching that selector exists in your `index.html`.
* **Resolution:** Verify that your `index.html` file contains a container element with the corresponding ID or class (e.g. `<div id="app"></div>`).

### AVX_R02
* **Message:** `Page "{pageName}" is not registered. Ensure page class is named correctly.`
* **Type:** Throw Exception (`AvenxError`)
* **Cause:** The router attempted to mount a page named `{pageName}` which was not registered with the application.
* **Resolution:** Verify that your page file (e.g. `src/pages/home.page.js`) is named correctly, exports a default class, and that your router config correctly maps the hash pattern to the exact name.

### AVX_R03
* **Message:** `Component "{compName}" is not registered. Registered components: {list}`
* **Type:** Throw Exception (`AvenxError`)
* **Cause:** A page or component template declared a custom child component tag (e.g. `<Navbar />`), but that component is not registered.
* **Resolution:** Verify that the component file exists in `src/components/` and is successfully parsed during the compile phase.

### AVX_R04
* **Message:** `Circular dependency detected in computed property "{propertyName}".`
* **Type:** Console Warning
* **Cause:** A computed property's evaluation expression reference-loops back to itself (e.g. computed `a` reads state proxy value `a`, which triggers computed `a` again).
* **Resolution:** Refactor the computed property to ensure it only depends on standard state fields or other non-circular properties. Do not modify or read the property itself inside its getter.

### AVX_R05
* **Message:** `Failed to evaluate computed property "{propertyName}". Expression: "{expression}". Error: {error}`
* **Type:** Console Warning
* **Cause:** An unhandled error occurred while executing the computed getter script block.
* **Resolution:** Fix syntax errors or reference errors (such as accessing properties of undefined values) within your computed property definition.

### AVX_R06
* **Message:** `Navigation guard denied transition to route "{hash}".`
* **Type:** Console Warning
* **Cause:** A route guard mapped to the route explicitly returned `false` inside its `canActivate()` method, blocking navigation.
* **Resolution:** This is expected behavior for authentication or access control. If unexpected, check the conditional checks inside your `.guard.js` file.

### AVX_R07
* **Message:** `Navigation guard threw an error during evaluation for route "{hash}": {error}`
* **Type:** Console Error Log (reverts navigation)
* **Cause:** A route guard's `canActivate()` method threw an unhandled promise rejection or error (e.g. failed fetch requests).
* **Resolution:** Wrap async calls in `try/catch` inside your route guards and return `false` or redirect paths instead of letting exceptions bubble out.

### AVX_R08
* **Message:** `Failed to render interpolation expression "{expression}". Error: {error}`
* **Type:** Console Warning
* **Cause:** An interpolation block `{{ ... }}` in a component's template failed to resolve (e.g. accessing a nested property of a null state field).
* **Resolution:** Check that all variables referenced inside `{{ ... }}` are correctly defined on `this.state` or `this.computed`.

### AVX_R09
* **Message:** `Event handler execution failed for statement "{statement}". Error: {error}`
* **Type:** Console Error Log
* **Cause:** An inline event handler attribute (e.g. `@click="increment()"`) threw an exception during callback execution.
* **Resolution:** Verify that the target method exists on your component class and is spelled correctly, and that parameters passed are defined in scope.

### AVX_R10
* **Message:** `Bridge "{name}" is already registered. Available bridges: {list}. Suggestion: {suggestion}`
* **Type:** Throw Exception (`AvenxError`)
* **Cause:** The application tried to register a bridge with a name that is already in use.
* **Resolution:** Ensure bridge names are unique. If you need to register a new bridge, choose a different name.

### AVX_R11
* **Message:** `State mutation detected during the update/render lifecycle. Avoid modifying component state inside templates, getters, computed property definitions, or lifecycle hooks like onUpdate.`
* **Type:** Throw Exception (`AvenxError`)
* **Cause:** Modifying properties in `this.state` or `this.props` during the rendering or updating lifecycle of the component (such as inside a computed property definition, getter, template interpolation expression, or lifecycle hooks like `onUpdate`).
* **Resolution:** Avoid synchronous state changes inside the update lifecycle. If you need to update state in response to an update, defer it to the next microtask (e.g. using `setTimeout` or `Promise.resolve().then()`), or compute the value using a computed property.
